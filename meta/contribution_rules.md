# Contribution Rules

## Contribution Guidelines

### Code Contributions
- Must be ARM64 assembly
- Must compile and run
- Must follow AAPCS64
- Must include tests

### Documentation
- Must cite references
- Must be accurate
- Must be clear
- Must be complete

### Quality Standards
- No placeholders
- Working code only
- Proper error handling
- Security best practices

## Review Process

1. Code must compile
2. Code must run correctly
3. Must follow ABI conventions
4. Must include documentation
5. Must cite references

## Reference Requirements

- All concepts must be cited to [`../references/`](../references/) directory
- Use official ARM documentation (see [`../references/external_links.md`](../references/external_links.md))
- Link to authoritative sources in `references/` directory
- No uncited knowledge - all ARM64 concepts must trace to `references/`

## Submission Format

- Follow existing structure
- Include build instructions
- Provide test cases
- Document changes
